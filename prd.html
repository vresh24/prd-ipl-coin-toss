Bistro Toss & Win

A match-time coin toss mini-game that rewards users with freebies and BOGOs to drive Bistro orders during IPL matches.

ProductBistro
TypeMini-game / Reward
AvailabilityMatch hours only
FrequencyOnce per match
Reward ExpiryEnd of day (7 days max)
🎯
Objective

Users always think of ordering from Bistro during a match. Order growth on match-relevant assortments (popcorn, fries, etc.) is already visible in the first hour — this feature doubles down on that behaviour.

💡
Solution

A lightweight coin toss game active only during match time. Low friction + good rewards = high perceived ROI. Users get Freebies or BOGOs that they can apply at checkout.

New Pages Introduced

🎮
Game Page
  • Today's winnable rewards shown upfront
  • Team city names for the match
  • Heads or Tails selector → Toss CTA
  • Win/lose animation + outcome
  • List of already-won rewards
  • FAQ section
  • Countdown to next game (post-play / off-hours)
🎁
Rewards Page
  • Entry: from Cart only
  • Shows Active, Used, Expired rewards
  • Sorted: Won today → recency
  • Unlimited offers can be applied per order
  • Multiple BOGOs can be used simultaneously
  • Selected rewards reflect on Cart
  • Empty state: "Tune in tomorrow to play and win"

Key Constraints

✅
CR = 0 Users Are Eligible
CR = 0 users can play the game. When a user transitions from CR=0 → CR=1, feed and all pages must update instantly to show game entry points (FAB animation, PT callout if applicable).
✅
Stackable with BISTRO50 & 50% Off
Game rewards (freebies and BOGOs) can be applied alongside BISTRO50 and any 50% off promo codes. No stacking restriction at cart.
📱
Scope: Current Feed Only
This feature is scoped to the current feed. New feed integration will be handled in a future phase.
📣
PT Banner — Always On During Off-Match Hours
The Promised Time (PT) banner for the upcoming game start is shown at all times during off-match hours — not just Week 1. After Week 1, PT reverts to normal grid during active match hours only.
Bistro Toss & Win › Game Logic

Game Construct & Rules

How the game works, win probabilities, and timing rules

Game Flow

1
Reveal Rewards — Show today's possible winnable rewards before the user plays
2
Show Match Context — Display city names of today's teams
3
Select Side — User picks Heads or Tails
4
Toss CTA appears — "Toss the Coin" button shown after selection
5
Backend pre-determines result — Result is saved to Rewards Page immediately on tap. Animation is purely visual.
6
Animation plays — Coin toss animation
7
Show outcome — Win: reward details + CTA to Feed. Lose: "Better luck next time, try tomorrow"

Win Probability Types

✅ Guaranteed Win

On some days, winning is guaranteed regardless of H/T choice. Messaging: "You will win BOGO on burgers OR free peri peri fries"

🎲 Chance-based (≤50%)

On other days, the correct guess wins. Messaging: "You stand a chance to win Free peri peri fries or nothing on wrong guess"

Timing Rules

📅 Play Frequency
  • Once per match per user
  • Double headers = 2 plays in a day
  • Single header window: 7 PM – 11:59 PM
  • Double header timing: TBD
⏰ Game LIVE Definition
  • Game is only LIVE during match hours
  • FAB shows coin animation when LIVE
  • FAB shows static coin when NOT LIVE
  • Off-hours: show countdown + past rewards

FAB (Floating Action Button) States

🪙✨
Animated Coin
Game is LIVE AND user hasn't played today's match
🪙
Static Coin
Game NOT LIVE OR user has already played today
➡️
Always tappable
Always redirects to Game Page regardless of state

Bistro Toss & Win › Rewards

Reward System

Types, expiry, application rules, and multi-BOGO handling

Reward Types

🎁 Freebie
  • Auto-added to cart if won today
  • Pop-up shown on auto-add
  • If removed → no re-auto-add
  • Unlimited freebies can be active and applied
🔁 BOGO
  • "Complete your offer" rail in cart
  • Multiple BOGOs can be used simultaneously
  • Manage BOGOs from Rewards Page (UX: TBD)
  • Eligible product cards show BOGO badge when won
💨 Nothing
  • Shown after losing toss
  • "Better luck next time"
  • "Try again tomorrow"
  • No cart impact

Expiry & Sorting Rules

⏳ Expiry
  • Rewards expire in 7 days from when given
  • Expiry is end of day (midnight)
  • Can be used any time before expiry
  • Different rewards each match day
📋 Display Sorting
  • Won today → shown first, tagged "Won today"
  • Then by recency (newest first)
  • Tagged "Expiring on [date]"
  • Used and Expired shown below Active

Selection & Stacking Rules

✅
Unlimited Offers per Order
Users can apply unlimited offers. Multiple BOGOs and freebies can all be active on the same order. UX for selecting multiple BOGOs is TBD.
✅
Stackable with BISTRO50 & 50% Off
Game rewards can be combined with BISTRO50 and any 50% off promo codes. No stacking restriction applies.
🔄
Freebie Auto-add Behaviour
If a freebie was won today, it is auto-added when user opens cart. If the user explicitly removes it, do NOT auto-add it again in that session.
Bistro Toss & Win › Edge Cases

Edge Cases

Solved edge cases and open questions that need resolution

Solved Edge Cases

✓ SOLVED App crash / internet drop during coin spin
Backend pre-determines and saves the result the moment "Toss" is tapped. Reward will be present in Rewards Page even if animation never completes. The frontend animation is purely cosmetic.
✓ SOLVED CR = 0 eligibility & CR = 0 → CR = 1 transition
CR = 0 users are eligible to play. When a user transitions from CR=0 to CR=1, the feed and all other pages must update instantly to show game entry points (FAB animation, PT callout if applicable).
✓ SOLVED User removes auto-added freebie
If user explicitly removes the auto-added freebie from cart, do NOT re-add it automatically. Respect user intent.

Additional Solved Cases

✓ SOLVED Multiple BOGOs
Users can use multiple BOGOs on a single order. UX for selecting and managing multiple BOGOs simultaneously is TBD — to be designed in a future sprint.
✓ SOLVED Freebie item is out of stock
Avoid this through proactive inventory planning. Offers will be designed around items with sufficient stock. No reactive fallback — prevention is the strategy.
✓ SOLVED Match ends early (rain / quick wickets)
Game runs for the entire scheduled match duration regardless of early finish. No early cutoff for rain or quick fall of wickets.
✓ SOLVED Match goes to Super Over
Game window is NOT extended for Super Overs. The game ends at the scheduled match end time. Super Over extension can be planned for V2.

Open Questions

⚠️ OPEN Order cancellation after offer applied
Scope: applies to cancellations where Bistro is at fault — PNA (product not available), rider mistakes, or other Bistro-side errors. What happens to the reward in these cases? Reinstate? Mark as used? Policy TBD.
